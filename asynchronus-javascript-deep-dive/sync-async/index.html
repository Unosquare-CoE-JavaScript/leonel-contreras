Syncronous vs Asyncronus

Syncronous code

Are codes that execute line to line, in orden to execute the next line, needs to finisih the prevous one. When a process is very complex could take a lot of time to finish, blocking other processes, this mean you can´t doing anything else unitl the process end, we called this blocking code.

Asyncronus code

Are codes that don´t need to wait to previous lines finish to execute the new ones, when we use Asyncronus we know that in some point in the time we'll get a response but we don´t know when, this kind of code it´s called | no-blocking code, the most of the time in JS we prefer to use it, but may exist some exceptions for example maybe we want to wait and don´t do anything else until user response.

Advantages and disadvantages

Synchronous code

    • Advantage: Easy to write and to reason about it
      Disadvantage: May create blocking code
      Disadvantage: Less performance

    • Asynchronus code

    • Advantage: very performant
      Advantage: Eliminates code blocking
      Disadvantage: It can be difficult to reason about
      Disadvantage: Harder to write
